## 1. 定位与范围

- 串口无刷电机是云台主执行器。
- 第一代和第二代张大头闭环步进电机仅作为备用执行器，不属于当前主云台控制链。
- 两台电机均使用 Emm 固件和 TTL 串口直连。本实现不使用第二代资料中的 X 固件协议，
  也未实现 STEP/DIR 后端。
- 第二代已完成烧录、串口回包、位置/速度/停止/失能和速度租约实机验证；第一代仍待接线验证。

## 4. 软件边界

```text
未来云台执行器仲裁
  -> ZdtStepper_Request* / ZdtStepper_Service
    -> ZdtProtocol_Build* / Parse*
      -> BSP_ZdtUart_*
        -> UART2/UART3 + DMA_CH1/DMA_CH2
```

- `app` 只调用设备层，不拼协议字节，也不直接调用 DriverLib。
- `module/device/zdt_protocol.*` 负责 Emm 帧、响应解析和加速度单位换算。
- `module/device/zdt_stepper.*` 是两路备用电机的唯一命令状态机。
- `bsp/source/bsp_zdt_uart.c` 只负责 UART/DMA/RX ring，不判断云台模式。
- 当前尚未实现统一无刷/步进执行器仲裁；因此应用没有调用
  `ZdtStepper_SelectBackupBackend()`，张大头保持锁定。

## 5. 默认安全行为

- `ZdtStepper_Init()` 后 `backend_selected=0`。
- 未显式选择备用后端时，不查询、不使能、不接受速度/位置/停止请求，也不会自动运动。
- 相同速度或位置帧只发送一次，不随 100 Hz `SystemTask` 重发。
- 单路最小发送间隔 20 ms；轮询间隔 50 ms；查询响应超时 20 ms，连续 3 次超时标记离线。
- 从备用后端退出时，两路依次发送立即停止和失能，完成后才清除退出状态。
- 这些保证只覆盖 MCU 运行时逻辑。调试 halt、MCU 失电、线缆故障和电机内部参数错误仍需
  依靠物理断电、机构限位和后续故障验收。

## 6. 两代差异

| 行为 | 第一代 Emm | 第二代 Emm |
| --- | --- | --- |
| UART | 独立 UART2 | 独立 UART3 |
| 固件版本回复 | 5 B | 7 B |
| 相对当前坐标模式 | 不允许 | 允许 |
| 位置命令运行中更新 | 拒绝并返回 busy | 允许新目标排队发送 |

第一代位置目标未到达时禁止连续覆盖，是为了避免旧固件不断重规划、长期处于低速加速段。
第二代仍受 20 ms 发送间隔和相同目标去重约束。

## 7. 加速度与稳定命令策略

Emm 的原始 `acc` 是 8 位非线性编码，不是直接的 `RPM/s`。设备层对上层暴露物理
`acceleration_rpm_s`，按手册关系换算：

```text
acc = 0     -> 直接启动
acc = 10    -> 约 81 RPM/s
500 RPM/s   -> acc 0xD8
```

初次板测建议从 500 RPM/s 开始，再根据负载、电流、机械冲击和到速时间调整。稳定策略是
“目标变化时发送一次，由电机内部轨迹执行”，不是以控制循环频率反复发送同一位置帧。

## 8. 已验证与未验证

已验证：

- Emm 命令编码、响应解析和加速度换算 host 单元测试。
- mock UART 状态机：默认零发送、相同速度不重发、第一代位置忙拒绝、第二代位置更新、
  退出时两路停止并失能。
- SysConfig 生成 0 errors（保留 1 个既有 ProjectConfig warning）。
- FreeRTOS 与 App full rebuild 均为 0 Error / 0 Warning。
- 第二代地址 `0x01`、115200 8N1 实机在线，固件原始版本值 `200`，硬件原始版本值
  `8970 (0x230A)`。
- 第二代相对当前位置 `+15 deg` 后实测 `+15.007 deg`，反向返回后的基准误差为
  `0.044 deg`；正反速度反馈分别为 `+29/+30 rpm` 和 `-29/-30 rpm`。
- 显式停止后速度均回到 `0 rpm`；1.5 s 速度租约在不发送 Stop 时自动停止，过期计数增加 1。
- 最终第二代 `enabled=false`、`motion_active=false`、速度 `0 rpm`、状态位 `0x02`，
  无效响应、第二代超时、堵转和堵转保护计数均为 0。

未验证：

- 第一代板上 UART、地址、波特率、回包和运动差异。
- 第二代断线恢复、带载电流、温升、堵转保护和机械限位。
- 与未来无刷主后端的互斥切换和故障降级。

## 9. 实机验收门禁

开始运动测试前必须同时满足：

1. 用户人在现场并明确许可本次运动。
2. 机构架空或与负载脱开，默认目标为零，物理断电随手可用。
3. 分别确认两台电机的 TTL 电平、共地、TX/RX、地址、115200 8N1 和 Emm 固件。
4. 先只接一台，验证查询回包；再执行低速、小角度、单次目标。
5. 由用户给出云台安全速度、角度范围和初始加速度，未确认前不做连续或大角度运动。
6. 记录 TX/RX、响应超时、实际速度、到达状态、电流和温升；第一代重点复核不会长期慢转。

只有以上板测完成，才能把状态从“软件验证通过”提升为“备用执行器实机可用”。
