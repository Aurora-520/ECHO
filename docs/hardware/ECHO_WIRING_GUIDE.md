本文是 ECHO 工程的长期接线入口，记录当前固件已有功能、正在开发的硬件接口和后续预留。
接线或代码变化后继续更新本文件，不依赖聊天记忆。表中的状态含义如下：

| 状态 | 含义 |
| --- | --- |
| 已验收 | 已有阶段构建、烧录和板上证据，可按对应条件使用 |
| 已配置，未验收 | 当前 Phase 2A 工作树已有代码/引脚配置，但尚未完成构建、烧录或板测 |
| 预留未实现 | 只有设计规划，当前固件不会使用该引脚或产生该功能 |
| deferred | 尚无可用物理后端或被硬件门禁阻止 |

## 2. 已验收低功率接线

### 2.1 DAPLink / SWD

```text
DAPLink                 天猛星 MSPM0G3507
SWDIO   --------------> PA19
SWCLK   --------------> PA20
nRESET  --------------> nRESET
GND     --------------> GND
```

- 同一个 DAPLink 不能同时被 Keil、OpenOCD 或另一个 GDB 会话占用。
- DAPLink 的 `3V3` 引脚是供电输出还是目标电压参考，必须按手中探针丝印/说明确认。
- 禁止把 DAPLink 的 3.3 V 输出并到已经由其他电源供电的 3.3 V 轨。
- 工程已验证目标识别、下载、校验、复位、断点和 Watch；异常发热的天猛星板继续禁用。

### 2.2 UART1 遥测与参数

```text
3.3 V USB-TTL / DAPLink UART       天猛星 MSPM0G3507
RXD       <----------------------- PA8  (UART1_TX)
TXD       -----------------------> PA9  (UART1_RX)
GND       ----------------------- GND
VCC       不接（主控已由其他电源供电时）
```

串口配置固定为 `230400 baud, 8 data bits, no parity, 1 stop bit, no flow control`。PA8/PA9 是
3.3 V 逻辑，禁止直接接 RS-232 电平或 5 V TTL 输出。COM 号由系统枚举，不能把 COM4/COM8
当成永久接线事实。网页遥测、采集脚本和串口助手不能同时独占同一个 COM 口。

### 2.3 SSD1306 OLED

| OLED 四针模块 | 天猛星 MSPM0G3507 |
| --- | --- |
| `VCC` | `3.3 V` |
| `GND` | `GND` |
| `SDA` | `PA0` |
| `SCL` | `PA1` |

当前驱动以 400 kHz 扫描地址 `0x3C` 和 `0x3D`，实物验收使用 `0x3C`。现用模块很可能自带
I2C 上拉；若更换模块，先确认是否已有上拉，缺少时再分别用约 4.7 kohm 将 SDA/SCL 上拉到
3.3 V，不能重复并联过多上拉。整车线束应使用短线、可靠共地并在显示模块附近加去耦。

OLED 显示 Overview、RTOS、COMM、DEVICE、PARAM 五页。当前切页和参数编辑使用 debugger
虚拟按键变量，不存在可直接接线的五个物理按键引脚。

### 2.4 板载心跳 LED

板载 LED 已连接到 `PB22`，不需要外接 LED。ServiceTask 收到 SystemTask 心跳后翻转该引脚；
它是运行指示，不是独立的执行器安全灯。

## 3. Phase 2A AT8236 与轮组

### 3.1 用户确认的模块端子映射

| 轮组 | D153B/AT8236 通道 | 电机线 `+` | 电机线 `-` | 编码器 A 相 | 编码器 B 相 |
| --- | --- | --- | --- | --- | --- |
| 左轮 | Motor A | `AO1 / AOUT1` | `AO2 / AOUT2` | `E1A` | `E1B` |
| 右轮 | Motor B | `BO1 / BOUT1` | `BO2 / BOUT2` | `E2A` | `E2B` |

用户原始说明中的 `A02` 在工程文档统一写为 `AO2`，即字母 `O`；接线前仍须按手中 D153B
模块丝印核对，避免把字母 `O` 与数字 `0` 混淆。

```text
左轮电机 +  ---------------- AO1       右轮电机 +  ---------------- BO1
左轮电机 -  ---------------- AO2       右轮电机 -  ---------------- BO2
左编码器 A  ---------------- E1A       右编码器 A  ---------------- E2A
左编码器 B  ---------------- E1B       右编码器 B  ---------------- E2B
```

这里的 `+/-` 仅表示电机线标记，不保证对应整车前进方向。左右轮镜像安装造成的符号差异，
后续通过软件中的 `motor_output_sign` 和 `encoder_count_sign` 归一化，不交换已确认的电机线或
A/B 相线。

### 3.2 当前左编码器代码接线门禁

当前 Phase 2A 工作树已把左编码器规划为：

```text
D153B E1A -- [5 V -> 3.3 V 电平转换，方案待确认] --> PA29 / TIMG8 QEI PHA
D153B E1B -- [5 V -> 3.3 V 电平转换，方案待确认] --> PA30 / TIMG8 QEI PHB
D153B GND ------------------------------------------ 主控 GND
```

D153B 的 `E1A/E1B/E2A/E2B` 是编码器信号直出，没有电平转换；模块电机接口给编码器提供
5 V，而 PA29、PA30、PB6、PB7 不是 5 V tolerant。首次连接前必须在 MCU 断开时测量高电平：

- 高电平高于 3.6 V 时，必须使用经过审查的 5 V 到 3.3 V Schmitt buffer/level shifter，或
  经过计算和波形验证的分压网络。
- 在电平转换方案冻结前，禁止将 `E1A/E1B` 直接连接到 `PA29/PA30`。
- 禁止把编码器 5 V、`AO1/AO2` 或其他电机输出误接 MCU GPIO。
- 用户确认实际编码器信号为 3.3 V。当前左编码器已完成 0/0 构建、烧录回读校验、正反手转、
  粗略一圈量级和 120 秒静止零漂移；该结论只覆盖无动力编码器，不覆盖 AT8236 或电机输出。

### 3.3 当前右编码器状态

右编码器固定为 `E2A -> PB6` 上升沿软件 x1、`E2B -> PB7` 方向输入。无动力板测确认整车向前
原始计数为负，因此右轮 `encoder_count_sign=-1`；不交换 E2A/E2B。右轮向前/向后粗略一圈
分别得到 `-18,632` / `+21,691`，只用于方向确认，CPR 仍采用 provisional 17,007。

单独测试右轮时左编码器未接，PA29/PA30 浮空会产生无效 QEI 计数和 issue 16。左右联合测试
必须同时接好两只编码器，不能把未接输入的 Health fault 误判为右轮故障。

### 3.4 当前仍禁止上动力的端子

四路电机输入 `PB8/PB9/PB12/PB13` 已配置为 10 kHz PWM，并完成 VM 断电、动力线断开条件下
的正负 5%/200 ms 逻辑命令板测。固件启动默认四路强制低，UART 命令只允许单电机、最高
10%、最长 500 ms，SystemTask 是唯一输出写入者。上述结果尚未用示波器测量实际频率/占空比，
也没有连接 D153B 输入或驱动电机，不能据此直接上 VM。

D153B 的 `VM`、逻辑电源、VREF、使能/待机、GND 和 4S 配电仍未形成完整已验收接线图。
首次带动力测试前还必须确认静止四路低、单轮架空、另一电机断开、限流值和物理断电。

## 4. 当前推荐的低功率调试组合

仅验证已验收 Phase 1F 功能时，可保持电机动力部分完全断开：

```text
PC
 |- DAPLink -> SWDIO PA19 / SWCLK PA20 / nRESET / GND
 |- 3.3 V USB-TTL -> RXD<-PA8 / TXD->PA9 / GND
 |
天猛星 MSPM0G3507
 |- 3.3 V / GND / PA0 / PA1 -> SSD1306 OLED
 |- PB22 -> 板载 LED（无需外接）
 |
 X  D153B VM / 4S / 电机动力：断开
 X  编码器相线：电平转换未确认前断开
```

若 DAPLink 同时提供 UART，可复用同一探针的串口功能，但 SWD 调试端和 COM 端仍是两个独立
软件资源。任何电源连接都只能在确认探针/适配器引脚性质后进行。

## 6. 通用上电安全检查

1. 先断开 USB、DAPLink、逻辑电源、电机电源和 4S，按实物丝印逐线核对。
2. 用万用表做通断、短路和极性检查，不仅依赖线色。
3. 选择唯一供电方案，禁止两个已供电的 3.3 V 电源轨直接并联。
4. 先验证主控、UART 和 OLED 低功率链路，再逐个加入传感器或驱动模块。
5. 涉及电机、轮组、云台或 4S 前，必须由用户在场明确许可；架空机构、默认零输出并准备
   物理断电。
6. 首次运动测试使用限流电源和最低可验证输出。异常发热、异味、噪声或失控时立即断电。

## 7. 文档更新规则

每次硬件、引脚或代码状态变化时：

1. 更新本文件的总表、对应章节和修订记录。
2. 写明信息来源：实物丝印、指定版本原理图、当前 SysConfig/BSP、用户确认或板测证据。
3. 代码只规划但未实现时标为“预留未实现”；代码存在但未板测时标为“已配置，未验收”。
4. 引脚变化时同步检查 SysConfig、BSP、Phase 文档和 Keil 工程，不手改 generated 文件。
5. 在 `docs/handoff/CURRENT_HANDOFF.md` 记录实际物理接线、供电、后台进程和下一步。
6. 阶段验收后再把相应条目标为“已验收”，并记录 commit/tag 和测试证据。
